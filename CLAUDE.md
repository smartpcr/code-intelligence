## Quick Context

`code-intelligence` is a Go monorepo with two services: `agent-memory`, a hybrid graph/episodic memory service for coding agents, and `clean-code`, a clean-code metrics, policy, and refactor-planning service. ## Notes

- Use service-local modules and Makefiles; there is no root `go.work`.
- Keep generated protobuf bindings in sync with proto edits.
- Preserve append-only/tombstone/retraction patterns in storage changes.
- Keep environment access centralized through `services\clean-code\internal\config` for `CLEAN_CODE_*` values.
- Current local validation caveats are documented in [Tests](.claude/context/tests.md).
